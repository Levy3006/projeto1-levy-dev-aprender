GIT INIT - cria um novo repositório do Git.(é o primeiro comando ao criar um novo projeto). git init serve também para converter um projeto não versionado em um repositório do Git. git init é uma maneia muito fácil de criar novos projetos controlados por versão!

GIT STATUS - serve para fornecer algumas informações importantes sobre a branch que você está usando, como nome da branch, commits feitos, etc, inclusive se a branch está atualizada em relação a master, e quais arquivos foram alterados!

GIT ADD "NOME DO ARQUIVO" - para incluir as alterações de um arquivo no próximo commit. este comando serve apenas para 1 arquivo. para adicionar todos os arquivos modificados de uma só vez, use o GIT ADD -A. (o GIT ADD não altera o repositório e as alterações não são salvas até usarmos o GIT COMMITT)

GIT COMMIT -m "descrição" - para criar uma nova versão do código.

GIT REFLOG - para vizualizar versões anteriores do código
GIT RESET --HARD numeroID - para restaurar uma versão anterior do repositório local.

GIT BRANCH  - mostra todas as branchs e a branch que você está printa destacada
GIT BRANCH + NOME - cria uma nova branch com o nome "NOME"
